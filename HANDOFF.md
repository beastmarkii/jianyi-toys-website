## 4. ★ i18n 시스템 (가장 중요)

**모든 화면 텍스트는 `i18n.js` 에 있습니다.** HTML의 한자는 JS 로드 전 잠깐 보이는 기본값일 뿐이며, 실제 표시는 `i18n.js` 사전이 덮어씁니다.

### 구조
```js
window.I18N = {
  zh: { "hero.title": "...", ... },   // 中文 (기본)
  en: { "hero.title": "...", ... },   // English
  ja: { "hero.title": "...", ... },   // 日本語
};
```

### HTML에서 사용
```html
<!-- 순수 텍스트 -->
<h2 data-i18n="about.title">关于我们</h2>

<!-- HTML 태그 포함 (strong, span, small 등) -->
<p data-i18n-html="about.p1">...<strong>15 年</strong>...</p>
```

### 텍스트를 바꾸려면
**HTML만 고치면 안 됩니다.** 반드시 `i18n.js` 의 **zh / en / ja 세 곳 모두** 수정하세요.
(HTML의 기본값도 같이 맞춰주면 더 좋습니다 — JS 로드 전 깜빡임 방지)

### 기타 동작
- `{year}` 플레이스홀더는 표시 시점에 현재 연도로 치환 (footer.copy)
- 페이지별 `<title>` 은 `script.js` 의 `PAGE_TITLE_KEY` 맵 + `<body data-page="...">` 로 결정
- 초기 언어: localStorage 저장값 → 브라우저 언어 자동감지 → 중문(기본)

## 5. ★ 캐시 버전 규칙 (필수, 자주 실수함)

모든 HTML이 CSS/JS를 `?v=N` 쿼리와 함께 불러옵니다.

```html
<link rel="stylesheet" href="styles.css?v=13" />
<script src="i18n.js?v=13"></script>
```

**CSS나 JS를 한 글자라도 수정하면 8개 HTML 전체의 버전을 올려야 합니다.**
안 올리면 브라우저·GitHub Pages 캐시 때문에 "수정이 반영 안 된다"는 현상이 생깁니다.
(이 프로젝트에서 이 문제로 여러 번 헤맸음)

cd WWW && sed -i 's/?v=13/?v=14/g' *.html

## 6. ★ 디자인 시스템 — 쿨 미니멀

사용자가 명품 스타일 4종 중 **② 쿨 미니멀 (Aesop·Bottega 계열)** 을 선택했습니다.

### 핵심 원칙
1. **모노크롬** — 색을 쓰지 않는다. 오프화이트 + 먹빛 블랙 + 회색뿐
2. **헤어라인** — 그림자·둥근모서리 대신 얇은 선(1px)으로 구획
3. **얇은 타이포** — 제목은 font-weight 300 (숫자는 200), 굵게 쓰지 않는다
4. **여백** — 섹션 패딩 크게(70~120px). 빽빽하면 저렴해 보인다
5. **대문자 트래킹 라벨** — 작은 라벨은 uppercase + letter-spacing .26em
6. **각진 모서리** — `--radius: 2px` (사실상 직각)

### 컬러 토큰 (`styles.css` 상단)
```css
--paper:#f0f0ed;   /* 배경 (쿨 오프화이트) */
--paper-2:#e8e8e4;
--surface:#f7f7f5; /* 카드 호버 */
--ink:#171614;     /* 본문/제목 (먹빛 블랙) */
--ink-2:#4b4a45;   /* 보조 텍스트 */
--mute:#8d8c85;    /* 라벨/캡션 */
--line:rgba(23,22,20,.14);   /* 헤어라인 */
--radius:2px;
```
다크 테마는 `:root[data-theme="dark"]` 에서 같은 토큰을 반전값으로 재정의합니다.

### 절대 하면 안 되는 것 (사용자가 명시적으로 거부한 것들)
| 거부된 것 | 사용자 표현 |
|---|---|
| 무지개/다색 아이콘 원형 | "고급스럽지가 않다" |
| 유아틱한 디자인 | "유아틱은 별로네" |
| 파스텔 그라데이션 배경, 글래스모피즘 | (이전 visionOS 디자인 폐기) |
| 딱딱한 박스+화살표 흐름도 | "너무 딱딱해" |
| 생산흐름 + 품질을 한 페이지에 합치기 | "가독성이 떨어진다" |

### 프로세스 표현 방식
생산 7단계는 **에디토리얼 흐름**으로 표현합니다 (`.process-flow`):
상단에 검정 실선 → 각 단계 시작점에 작은 점(●) → 큰 얇은 숫자(01~07) → 단계명.
박스·화살표·아이콘·애니메이션 없음.

**동작**: 히어로 로고 복제본이 화면 중앙에서 **고요히 페이드인 → 잠시 머묾 → 좌측 상단 헤더 로고 자리로 미끄러져 이동 → 스며들며 소멸**. 착지 시점에 실제 헤더 로고가 나타남.
